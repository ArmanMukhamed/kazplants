# PROJECT PLAN: Kazplants MVP

## 1. Цель проекта
Создать MVP интернет-магазина Kazplants для B2C-продаж растений по всему Казахстану с максимально простой и быстрой в поддержке архитектурой.

Функциональный фокус MVP:
- просмотр каталога и карточек товаров;
- поиск товаров;
- навигация по категориям;
- заказ через WhatsApp (без корзины и оплаты на сайте);
- управление товарами через Django Admin.

## 2. Технологическая архитектура MVP

### Backend
- Django (монолит)
- Django Admin
- Django Templates
- SQLite (на старте)

### Frontend
- Django templates
- Bootstrap 5 (или минимальный адаптивный CSS)

### Хранилище файлов
- `media/` для изображений категорий/товаров
- `static/` для CSS/JS/иконок

### Локализация (с заделом на масштабирование)
- включить i18n в Django;
- поддержка интерфейса: RU / KZ / EN;
- основной контент на первом этапе: RU;
- подготовка структуры для дальнейшего перевода товарного контента.

## 7. WhatsApp-оформление заказа

Кнопка: `Купить через WhatsApp`

Генерируемое сообщение:
- Здравствуйте!
- Хочу заказать товар: `[Название товара]`
- Цена: `[Цена]`
- Ссылка: `[URL товара]`

Технически:
- номер WhatsApp хранится в настройках проекта (`settings.py`, переменная наподобие `WHATSAPP_PHONE`);
- URL формируется через `https://wa.me/<phone>?text=<encoded_message>`.

## 8. SEO и URL-стратегия
- `slug` для категорий и товаров
- поля `meta_title` и `meta_description` у товара
- clean URL
- подготовка к sitemap (структура URL и возможность добавить `sitemap.xml` на следующем шаге)

- URL и базовые SEO-поля настроены
